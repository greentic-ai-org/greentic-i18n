PR-06: GitHub Actions automation (PR gate + bot PR updates)
Goal

Keep translations continuously up-to-date without humans doing busywork.

Workflows

i18n_validate.yml (PR check)

On PRs that touch any i18n/*.json:

run greentic-i18n-translator validate --langs <supported>

Run staleness/status checks only when:

i18n/en.json changed, or

.i18n/translator-state.json changed

i18n_translate_bot.yml (scheduled + label-trigger)

Nightly schedule

Also triggered by label i18n:translate

Runs translate for all langs (or tiered)

Commits changes

If no diffs, no-op and exit cleanly.

If diffs exist, create a fresh PR branch per run:

bot/i18n-translate/YYYYMMDD-HHMM

Open PR “i18n: update translations”

Notes

For CI, use API key mode (secrets) even if local dev uses browser login.

Keep local dev flow browser-based; CI is key-based.

Bot runs keep manual overrides by default; use --overwrite-manual only when explicitly forcing regeneration.
